# Contributing to VeridianAI

Thanks for your interest in contributing! Here's how to get involved.

## Before You Start
VeridianAI is free for personal and accessibility use, with a paid
tier for commercial deployment. By submitting a contribution, you
agree to the terms in LICENSE, including the grant of rights
covering both free and commercial use of your contribution.


## How to Contribute
1. Check open issues or open a new one describing your idea before starting
   significant work, this avoids duplicated effort.
2. Fork the repo and create a feature branch (`git checkout -b feature/your-idea`).
3. Follow existing code style and structure where possible.
4. Test your changes locally before submitting.
5. Open a pull request using the provided template.

## Reporting Bugs
Please use the Bug Report issue template and include:
- Steps to reproduce
- Expected vs. actual behavior
- Environment details (OS, Python version, hardware if relevant)

## Accessibility Contributions
Given VeridianAI's WCAG 2.2 AA compliance commitment, any UI/UX contributions
should maintain or improve accessibility. Screen reader testing (NVDA/JAWS)
is appreciated but not required to submit a PR, we'll help verify.

## Code of Conduct
This project follows our Code of Conduct. Please be respectful and
constructive in all interactions.
